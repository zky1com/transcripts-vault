# Naval - Before optimizing a system eliminate unnecessary requirements first then remove unnecessary parts —...

## Position

Musk's engineering methodology, as described in the Elon book: the correct order is (1) question every requirement and find the individual behind it, eliminate unnecessary requirements; (2) remove as many parts as possible while fulfilling the remaining requirements; (3) only then optimize how you build the remaining parts; (4) only then pursue cost efficiencies. The classic mistake is optimizing step 4 while leaving unnecessary requirements and parts in place. The Tesla battery mat example illustrates: teams were optimizing robot gluing speed when the requirement for the mat itself was obsolete.

## Evidence Cited

- Elon's method from the Jorgensson book: track down the individual behind each requirement, question whether it's necessary, eliminate it, then reduce parts, then optimize
- Tesla battery mat example: teams tried to speed up robot gluing; actual solution was eliminating the requirement because the original rationale (noise/heat) was obsolete
- "You first go and you track down the requirement... eliminate the requirement and then once you've eliminated the requirements that are unnecessary then you have a smaller number of requirements. Now you have parts and you try to get rid of as many parts as you can"

## Logic Chain

1. Every part and every requirement in a system was added by a specific person for a specific reason
2. Over time, the original reason may become obsolete while the part/requirement persists due to inertia
3. Optimizing how efficiently you manufacture an unnecessary part is wasteful
4. Therefore: before optimization, audit every requirement for continued necessity
5. This produces systems with fewer parts, each of which serves an active need â€” these systems are also cheaper, more reliable, and easier to improve
