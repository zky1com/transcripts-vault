# Naval - Bayesianism is a modern justificationist error — increasing confidence in a theory over time confuses corroboration with truth and cannot generate new theories

## Position

Bayesianism appears to be a sophisticated improvement on naive justificationism, but Naval (following Deutsch) argues it commits the same fundamental error. The Bayesian says: I collect evidence, I update my priors, and over time I become more and more confident in my theory. The confidence is treated as tracking truth — high confidence means the theory is probably true. But this confuses corroboration (the theory has survived many tests) with truth (the theory is correct). No amount of confirming evidence proves a theory true; it only shows it hasn't been refuted yet. Furthermore, Bayesianism cannot generate new theories. It can update weights among existing hypotheses in a finite hypothesis space, but it has no mechanism for producing a hypothesis that wasn't there before. The creation of new knowledge requires creativity, not probability-updating. Bayesianism, like justificationism, mistakes a useful tool for a complete epistemology.

## Logic Chain

1. Bayesianism: collect evidence → update priors → increase confidence → high confidence tracks truth
2. Problem 1: increasing confidence ≠ truth — theory could be highly corroborated and still wrong (black swan problem)
3. Problem 2: Bayesian updating requires a pre-existing hypothesis space — it cannot generate new hypotheses
4. Knowledge creation requires creative guessing, not updating across a fixed set of options
5. Bayesianism is a valid tool within constrained known hypothesis spaces (medicine, Monty Hall)
6. It fails as a general epistemology because it cannot create new knowledge, only redistribute confidence among existing guesses
